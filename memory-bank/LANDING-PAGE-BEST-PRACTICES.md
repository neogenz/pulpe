> Document compilé à partir de recherches UX/marketing 2025-2026, appliqué au contexte Pulpe.

## 1. Hero Section

### Règle d'Or
Le Hero doit répondre à 3 questions en moins de 5 secondes :
1. **Qu'est-ce que c'est ?** (catégorie produit)
2. **C'est pour qui ?** (identification)
3. **Pourquoi je devrais m'en soucier ?** (bénéfice)

### Structure Optimale

```
[Hook émotionnel / Pain point]  ← Crée la tension
[Headline principal]            ← La promesse
[Sous-titre]                    ← Comment on tient la promesse
[CTA primaire] [CTA secondaire] ← L'action
[Visual / Screenshot]           ← La preuve
```

### Headlines qui Convertissent

| Type | Exemple | Efficacité |
|------|---------|------------|
| **Résultat chiffré** | "Close Deals 30% Faster" | +300% vs vague |
| **Pain point résolu** | "Finis le stress de fin de mois" | Connexion émotionnelle |
| **Transformation** | "De chaos à clarté" | Avant/Après implicite |

**À éviter :** Headlines vagues type "Unleash Synergy" ou "The Future of X".

### CTA Best Practices

- **Contraste visuel** : +34% conversion avec bouton contrastant
- **Verbe d'action à la 1ère personne** : "Créer mon budget" > "Créer un budget"
- **Éviter** : "Learn More", "Submit", "Click Here"
- **Bon** : "Essayer gratuitement", "Commencer maintenant", "Voir la démo"

## 2. Psychologie de Conversion

### Le Cerveau "Crocodile" (Système 1)

Le visiteur scanne d'abord avec son cerveau primitif. Il cherche :
- **Danger/Douleur** : "Est-ce que ça résout MON problème ?"
- **Récompense** : "Qu'est-ce que j'y gagne ?"
- **Effort** : "C'est facile ?"

**Implication :** Présenter le pain point AVANT la solution.

### Framework StoryBrand (Donald Miller)

```
1. Le héros (l'utilisateur) a un PROBLÈME
2. Il rencontre un GUIDE (ton produit)
3. Le guide lui donne un PLAN
4. Et l'appelle à l'ACTION
5. Ce qui l'aide à ÉVITER L'ÉCHEC
6. Et à ATTEINDRE LE SUCCÈS
```

**Application Pulpe :**
- Héros : Personne stressée par ses finances
- Problème : Ne pas voir venir les grosses dépenses
- Guide : Pulpe (avec sa vue annuelle)
- Plan : 3 étapes simples
- Éviter : Le stress, les mauvaises surprises
- Succès : Sérénité financière

### Charge Cognitive (Nielsen Norman Group)

- **Limite** : 7 ± 2 éléments en mémoire de travail
- **Règle** : Une page = Un objectif = Un CTA principal
- **Sections** : 5-7 sections max avant fatigue cognitive

**Implication :** Pas plus de 3-4 features clés à mettre en avant.

### Effet "Pied-dans-la-Porte"

Demander un petit engagement d'abord augmente la probabilité d'un engagement plus grand ensuite.

**Application :** "Essayer gratuitement" (petit) avant "Créer un compte" (plus grand).

**Sources :** StoryBrand (Donald Miller), Nielsen Norman Group, Kahneman (Système 1/2)

## 3. Structure de Page

### Framework Problem-Agitate-Solve (PAS)

```
PROBLEM  → Identifier le problème ("Tu connais cette sensation...")
AGITATE  → Amplifier la douleur ("Et ça te coûte X...")
SOLVE    → Présenter la solution ("Avec Pulpe, tu...")
```

### Ordre Optimal des Sections

```
1. Hero (avec hook émotionnel)     ← Attention + Identification
2. Pain Points                      ← Validation du problème
3. Solution / Proposition de valeur ← Le "quoi"
4. Features (2-4 max)               ← Le "comment"
5. Social Proof                     ← Confiance
6. How It Works (optionnel)         ← Réduction de friction
7. Objections / FAQ (optionnel)     ← Lever les doutes
8. Final CTA                        ← Conversion
```

### Longueur Idéale

| Type de produit | Longueur recommandée |
|-----------------|---------------------|
| Produit simple/gratuit | Court (3-5 sections) |
| SaaS payant | Moyen (5-7 sections) |
| B2B complexe | Long (7-10 sections) |

**Pour Pulpe (gratuit, simple)** : 5-6 sections seraient suffisantes.

## 4. Social Proof

### Types par Efficacité

1. **Résultats chiffrés** : "30% d'économies en moyenne" (le plus puissant)
2. **Témoignages avec photo** : Visage = confiance
3. **Logos clients** : Pour B2B surtout
4. **Nombre d'utilisateurs** : "10,000+ utilisateurs"
5. **Notes/Avis** : Étoiles App Store

### Stats Clés

- 54% des landing pages SaaS n'ont PAS de témoignage (opportunité manquée)
- Témoignages avec photo : +35% de confiance perçue
- Résultats chiffrés : +2x plus convaincants que témoignages génériques

### Pour un Produit Jeune (comme Pulpe)

Alternatives au social proof classique :
- **Open source** : Transparence = confiance
- **Histoire du fondateur** : Authenticité
- **Données protégées** : Rassurance
- **Hébergement Europe** : RGPD = sérieux

**Sources :** [Unbounce State of SaaS](https://unbounce.com/conversion-rate-optimization/the-state-of-saas-landing-pages/)

## 5. Mobile First

### Statistiques 2025-2026

- **60%+** du trafic vient du mobile
- **53%** des visiteurs quittent si le chargement > 3 secondes
- **Touch targets** : Minimum 48px pour les boutons

### Checklist Mobile

- [ ] CTA visible sans scroller (above-the-fold mobile)
- [ ] Boutons assez grands pour le pouce (48px min)
- [ ] Pas de pop-ups intrusifs
- [ ] Images optimisées (WebP, lazy loading)
- [ ] Texte lisible sans zoom (16px min)
- [ ] Espacement tactile suffisant entre éléments cliquables

**Sources :** [Design Studio UI/UX](https://www.designstudiouiux.com/blog/saas-landing-page-design/), Google Core Web Vitals

## 6. Copywriting

### Principes de Base

| Règle | Exemple Mauvais | Exemple Bon |
|-------|-----------------|-------------|
| **Bénéfice > Feature** | "Vue calendrier 12 mois" | "Vois toute ton année d'un coup" |
| **Toi > Nous** | "Notre app permet de..." | "Tu peux..." |
| **Concret > Abstrait** | "Optimise tes finances" | "Sais exactement combien dépenser" |
| **Court > Long** | Paragraphes de 5 lignes | Phrases de 10-15 mots max |

### Mots Puissants (Trigger Words)

**Urgence/Rareté :** maintenant, aujourd'hui, limité
**Simplicité :** simple, facile, en 2 clics, 3 minutes
**Sécurité :** gratuit, sans engagement, garanti
**Émotion :** enfin, fini, sereinement, tranquille

### Formule Headline

```
[Résultat désiré] + [Délai/Facilité] + [Sans objection principale]

Exemple : "Reprends le contrôle de ton budget en 3 minutes. Sans prise de tête."
```

## 7. Tendances 2026

### Ce Qui Fonctionne

1. **Hyper-personnalisation** : Contenu adapté au segment visiteur
2. **Screenshots réels** : Pas de mockups génériques
3. **Micro-interactions** : Animations subtiles au scroll
4. **Design minimaliste** : Moins de texte, plus d'impact
5. **Dark mode option** : Attendu par les utilisateurs

### Ce Qui Ne Fonctionne Plus

1. **Stock photos génériques** : Détectées comme "fake"
2. **Sliders/Carousels** : Personne ne les utilise
3. **Pop-ups agressifs** : Augmentent le bounce rate
4. **Texte mur** : Personne ne lit les pavés
5. **"We are the leading..."** : Claims vides

**Sources :** [SaaS Hero Trends 2026](https://www.saashero.net/content/top-landing-page-design-trends/), [LandingPageFlow](https://www.landingpageflow.com/post/best-cta-placement-strategies-for-landing-pages)

## 8. Métriques à Suivre

### KPIs Essentiels

| Métrique | Bon | Excellent |
|----------|-----|-----------|
| **Bounce Rate** | < 50% | < 35% |
| **Time on Page** | > 1 min | > 2 min |
| **Scroll Depth** | > 50% | > 75% |
| **CTA Click Rate** | > 2% | > 5% |
| **Conversion Rate** | > 2% | > 5% |

### Pour A/B Testing

- **Volume minimum** : 100+ conversions par variante
- **Durée minimum** : 2 semaines (cycles hebdo)
- **Tester une variable à la fois** : Headline OU CTA OU image

## 9. Checklist Finale

### Avant Lancement

- [ ] Headline clair en < 5 mots
- [ ] Un seul objectif par page
- [ ] CTA contrastant et visible
- [ ] Mobile responsive testé
- [ ] Temps de chargement < 3s
- [ ] Screenshots réels du produit
- [ ] Pas de jargon technique

### Optimisation Continue

- [ ] Analytics configurés (scroll, clics, conversions)
- [ ] Heatmaps pour comprendre le comportement
- [ ] A/B tests réguliers (1 par mois minimum)
- [ ] Feedback utilisateurs collecté
